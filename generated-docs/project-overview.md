# Project Overview

## プロジェクト概要
- YM2151シンセサイザーのイベントログを、ターミナル上で編集・可視化するためのRust製ツールです。
- イベント間のタイミング調整や内容確認を支援し、JSON形式のイベントデータを直感的に操作できます。
- シンプルなインターフェースを通じて、YM2151のイベントデータ管理を効率化することを目的としています。

## ファイル詳細説明

- **Cargo.lock**: プロジェクトの正確な依存関係のバージョンを記録し、再現可能なビルドを保証します。
- **Cargo.toml**: Rustプロジェクトの設定ファイルで、プロジェクト名、バージョン、依存関係などが定義されています。
- **IMPLEMENTATION_PLAN.md**: 詳細な実装計画と今後のロードマップが記載された開発者向けのドキュメントです。
- **LICENSE**: 本プロジェクトのライセンス情報（著作権、利用条件など）が記載されています。
- **README.ja.md**: プロジェクトの概要、機能、インストール方法、使用方法などが日本語で説明された主要なドキュメントです。
- **README.md**: プロジェクトの概要、機能、インストール方法、使用方法などが英語で説明された主要なドキュメントです。
- **generated-docs/**: 生成されたドキュメントやレポートを格納するためのディレクトリです。
- **googled947dc864c270e07.html**: Google Search Consoleによるサイト所有権確認のために使用されるファイルです。
- **issue-notes/**: 開発中の特定の課題や検討事項に関するノート（開発者向け情報）を格納するディレクトリです。
  - `19.md`, `20.md`, `21.md`: 個別の課題に関する詳細なメモや情報が記述されています。
- **src/**: プロジェクトのソースコードが格納されているディレクトリです。
  - `app.rs`: アプリケーション全体の状態管理と主要なロジックを定義します。ユーザー入力の処理やUIの更新を統括します。
  - `event_editor.rs`: YM2151イベントの挿入、削除、値の編集など、具体的な編集操作に関するロジックを扱います。
  - `file_io.rs`: YM2151イベントログのJSONファイルからの読み込みと、JSONファイルへの保存処理を担当します。
  - `main.rs`: プログラムのエントリーポイントであり、初期設定、ターミナルの準備、イベントループの開始などを行います。
  - `models.rs`: YM2151イベント（`Ym2151Event`）やイベントログ全体（`Ym2151Log`）のデータ構造を定義します。
  - `navigation.rs`: ターミナルUI上でのイベントリストの上下移動など、ナビゲーションに関するロジックを管理します。
  - `preview.rs`: 編集中のイベントログのオーディオプレビュー機能（「P」キー操作）に関連するロジックを実装します。
  - `time_display.rs`: 累積時間表示と絶対タイムスタンプ表示の切り替え、およびその表示フォーマットに関するロジックを扱います。
  - `ui.rs`: アプリケーションのユーザーインターフェース（TUI）の描画ロジックを定義します。`ratatui`を使用して画面要素をレンダリングします。
- **test_data/**: テスト用のサンプルJSONデータを格納するディレクトリです。
  - `minimal.json`: 最小限のイベントデータを含むテストファイルです。
  - `sample.json`: より多くのイベントデータを含むテストファイルです。
