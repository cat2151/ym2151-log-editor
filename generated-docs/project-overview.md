# Project Overview

## プロジェクト概要
- YM2151シンセサイザーのイベントログを編集・可視化するターミナルユーザーインターフェース (TUI) アプリケーションです。
- イベントのタイミング調整や検査に焦点を当て、JSON形式のYM2151イベントデータを直感的に操作できます。
- Rustで開発されており、安定したパフォーマンスとターミナル環境での快適な操作性を提供します。

## 技術スタック
- フロントエンド: **ratatui** (ターミナルUIフレームワークで、リッチなテキストベースのユーザーインターフェースを構築), **crossterm** (ターミナルへの低レベルなクロスプラットフォーム操作を提供し、ratatuiのバックエンドとして動作)
- 音楽・オーディオ: (YM2151イベントログ自体が音楽データの一部ですが、直接的な音楽処理ライブラリは使用されていません)
- 開発ツール: **Rust** (安全性、並行性、パフォーマンスに優れたシステムプログラミング言語), **cargo** (Rustの公式ビルドシステムおよびパッケージマネージャ)
- テスト: (特定のテストフレームワークの明示的な記述はありませんが、Rustの標準テスト機能が利用されています)
- ビルドツール: **cargo** (Rustプロジェクトのビルド、依存関係管理、テスト実行などを行う統合ツール)
- 言語機能: **Rust** (メモリ安全性と高パフォーマンスを特徴とし、開発効率を高める多くの言語機能を提供します。また、**serde**によるシリアライゼーション/デシリアライゼーションと、そのJSON実装である**serde_json**を活用しています。)
- 自動化・CI/CD: (特筆すべき自動化・CI/CDツールは記述されていません)
- 開発標準: (特筆すべき開発標準ツールは記述されていません)

## ファイル詳細説明
- **`.gitignore`**: Gitのバージョン管理システムで追跡対象から除外するファイルやディレクトリを指定します。
- **`Cargo.lock`**: `Cargo.toml`で指定された依存関係の正確なバージョンを記録し、再現性のあるビルドを保証します。
- **`Cargo.toml`**: Rustプロジェクトのマニフェストファイル。プロジェクト名、バージョン、著者、依存関係、ビルド設定などを定義します。
- **`IMPLEMENTATION_PLAN.md`**: プロジェクトの詳細な実装計画、今後のロードマップ、および開発目標が記述されています。
- **`LICENSE`**: プロジェクトのライセンス情報（著作権や利用条件）が記載されています。
- **`README.ja.md`**: プロジェクトの日本語での概要、機能、インストール方法、使用方法などが説明されています。
- **`README.md`**: プロジェクトの英語での概要、機能、インストール方法、使用方法などが説明されています。
- **`generated-docs/`**: 通常、自動生成されたドキュメントや関連ファイルが格納されるディレクトリです。
- **`googled947dc864c270e07.html`**: Googleサービスによるウェブサイトの所有権確認に使用されるファイルです。
- **`issue-notes/`**: 開発中の課題、バグ報告、または改善点のメモを記録するためのディレクトリです。
    - **`19.md`, `20.md`, `21.md`**: 個別の課題やメモに関する詳細が記述されています。
- **`src/`**: プロジェクトの主要なソースコードが格納されているディレクトリです。
    - **`app.rs`**: アプリケーションの全体的な状態管理と主要なロジックを定義します。
    - **`event_editor.rs`**: YM2151イベントデータの編集に関する機能やロジックを扱います。
    - **`file_io.rs`**: JSON形式のYM2151イベントログファイルの読み込みと保存を処理します。
    - **`main.rs`**: アプリケーションのエントリーポイントであり、初期設定とメインのイベントループを管理します。
    - **`models.rs`**: YM2151イベント（`Ym2151Event`）やイベントログ（`Ym2151Log`）といったデータ構造を定義します。
    - **`navigation.rs`**: ターミナルUIにおけるイベント間の移動（上下キー操作など）のロジックを実装します。
    - **`preview.rs`**: YM2151イベントログ全体の再生プレビュー機能に関連する処理を実装します。
    - **`tests/`**: アプリケーションのテストコードを格納するディレクトリです。
        - **`app_tests.rs`**: `app.rs`で定義されたロジックに対するテストが含まれます。
        - **`mod.rs`**: テストモジュールを整理するためのファイルです。
        - **`model_tests.rs`**: `models.rs`で定義されたデータ構造に対するテストが含まれます。
    - **`time_display.rs`**: イベントの時刻表示モード（累積時間 vs 絶対タイムスタンプ）の切り替えと関連ロジックを扱います。
    - **`ui.rs`**: `ratatui`を使用して、ターミナルユーザーインターフェースの描画ロジックを定義します。
- **`test_data/`**: アプリケーションのテストに使用されるサンプルJSONデータファイルを格納するディレクトリです。
    - **`minimal.json`**: 最小限のイベントデータを含むテストファイルです。
    - **`sample.json`**: より多くのイベントデータを含むサンプルテストファイルです。
